# ADB Менеджер (Local ADB Manager)

Android-приложение, которое устанавливает и удаляет APK на этом же устройстве через
встроенную «Отладку по Wi-Fi» (Wireless Debugging, Android 11+) — без root и без компьютера
в повседневном использовании.

## Что уже готово (эта версия)

- Полная инфраструктура подключения: генерация ключа, сопряжение по 6-значному коду,
  автоматическое переподключение при следующих запусках (код вводится один раз).
- Поиск сервиса сопряжения в сети — вводить IP/порт вручную не нужно.
- Установка одного APK через системный файловый менеджер.
- Кнопка быстрого перехода в настройки разработчика.

## Что добавится следующим шагом

- Список установленных приложений с удалением в один тап (метод `uninstallPackage`
  и `listInstalledPackages` в `AdbRepository.kt` уже реализованы — осталось добавить экран).
- Фоновая служба для более стабильного соединения.

## Как собрать APK

Компилировать код может только Android Studio (или Gradle с Android SDK) — у меня в этой
песочнице нет ни того, ни другого, поэтому итоговый .apk я не могу собрать сам. Дальше — на
вашей стороне, но это буквально пара кликов:

1. Установите **Android Studio** (бесплатно): https://developer.android.com/studio
2. Откройте эту папку через **File → Open**.
3. Дождитесь окончания синхронизации Gradle (идёт автоматически, требует интернет —
   library и все зависимости скачаются сами).
   - Если Android Studio предложит создать/обновить Gradle wrapper — соглашайтесь, это нормально.
   - Если Android Studio предложит обновить версию Gradle/AGP — тоже можно соглашаться.
4. Подключите телефон по USB (с включённой обычной USB-отладкой) и нажмите **Run ▶**.
   Дальше USB уже не понадобится — только для этой самой первой установки.
   - Либо: **Build → Build Bundle(s) / APK(s) → Build APK(s)**, затем перекиньте готовый
     .apk на телефон любым способом и установите вручную.

## Сборка через GitHub Actions (без компьютера, без тяжёлых установок на телефон)

В проекте уже лежит `.github/workflows/build.yml` — он автоматически соберёт debug-APK
в облаке при каждой отправке кода на GitHub.

1. Зарегистрируйтесь на https://github.com (если ещё нет аккаунта) — можно прямо с телефона.
2. Создайте новый **пустой** репозиторий: **+** → **New repository** → задайте имя
   (например, `local-adb-manager`) → **Create repository**. Не добавляйте README/gitignore
   на этом шаге — репозиторий должен остаться пустым.
3. Загрузите файлы проекта одним из способов:

   **Способ А — через браузер (без Termux и без git):**
   Распакуйте архив на телефоне (файловый менеджер → «Извлечь»). На странице пустого
   репозитория нажмите **uploading an existing file**, затем через выбор файлов добавьте
   содержимое папки `LocalADBManager`. Если браузер не даёт загрузить папки целиком —
   загружайте по одной подпапке за раз (`app`, `.github` и т.д.), сохраняя пути.

   **Способ Б — через Termux (`git`, надёжнее сохраняет структуру папок):**
   ```
   pkg install git -y
   cd путь/к/распакованному/LocalADBManager
   git init
   git remote add origin https://github.com/ВАШ_ЛОГИН/local-adb-manager.git
   git add .
   git commit -m "Первая версия"
   git branch -M main
   git push -u origin main
   ```
   При push попросит логин — используйте Personal Access Token вместо пароля
   (GitHub → Settings → Developer settings → Personal access tokens → Generate new token,
   права достаточно `repo`).

4. Откройте вкладку **Actions** в репозитории — сборка запустится автоматически
   (или нажмите **Run workflow**, если запускаете вручную). Занимает пару минут.
5. После завершения (зелёная галочка) откройте сборку → в самом низу раздел
   **Artifacts** → скачайте `app-debug` (это zip с APK внутри).
6. Распакуйте, скопируйте `app-debug.apk` на телефон и установите (разрешите установку
   из этого источника, если Android спросит).

Если сборка упадёт (красный крестик) — откройте лог шага **Собрать debug APK** и пришлите
мне текст ошибки, поправим workflow или код под конкретную проблему.

## Как пользоваться

1. При первом запуске приложение попросит включить «Отладка по Wi-Fi»
   (кнопка в приложении откроет нужный раздел настроек) и открыть «Сопряжение по коду».
2. Введите 6-значный код, который покажет телефон. Приложение само найдёт адрес в сети.
3. При первом подключении система покажет стандартный диалог Android «Разрешить отладку
   с этого компьютера?» — нажмите **Разрешить** (лучше сразу отметить «всегда»). Это
   стандартное поведение ADB, обойти его нельзя и не нужно — это защита от чужих подключений.
4. Дальше при каждом запуске приложение подключается само, без вопросов.
5. На главном экране — кнопка «Выбрать и установить APK».

## Важно знать

- **Библиотека ADB не проходила аудит безопасности** (это прямо указано в её документации).
  Для личного использования на своём устройстве риск минимален, но это не production-grade
  крипто-библиотека уровня банковского приложения.
- Код не прогонялся через реальную компиляцию (в песочнице нет Android SDK), поэтому при
  первой сборке возможны мелкие ошибки. Самая вероятная — Android Studio не найдёт импорт
  `io.github.muntashirakon.adb.AbsAdbConnectionManager` в `LocalAdbConnectionManager.kt`:
  это будет значить, что у библиотеки чуть иначе называется пакет. Лечится за 10 секунд —
  клик на подчёркнутую строку → Alt+Enter → Android Studio сама предложит верный импорт.
- Резервное копирование данных приложения отключено намеренно (`allowBackup="false"`) —
  чтобы приватный ключ ADB не улетал в облачный бэкап.

## Технические детали (для справки)

| Компонент | Выбор | Почему |
|---|---|---|
| Протокол ADB / пейринг | [libadb-android](https://github.com/MuntashirAkon/libadb-android) 3.1.1 | Единственная найденная библиотека, которая реализует SPAKE2-пейринг именно на Android (не desktop-JVM) и явно поддерживает подключение к тому же устройству. На ней построен реальный опубликованный App Manager того же автора. |
| Сертификат ключа | BouncyCastle | Чтобы не использовать скрытые `sun.security` API Android. |
| Передача APK | Прямая запись во `getExternalFilesDir()` | Раз мы на том же устройстве — можно просто положить файл туда, куда `adbd` имеет доступ, и не реализовывать бинарный sync-протокол ADB (push) самостоятельно. |
| minSdk | 30 (Android 11) | Именно в Android 11 появилось сопряжение по коду. |

